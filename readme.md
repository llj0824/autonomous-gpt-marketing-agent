# Autonomous Marketing Agent

## Overview 

This project aims to build an automonous agent that automates posting content subtle marketing purposes. The agent is designed to streamline the process of identifying and ranking meaningful video segments, allowing for efficient content curation while maintaining manual control over the final selection.

## Features

1. **Channel Monitoring**: The agent monitors a predefined list of YouTube channels that you manually select to ensure relevance.
2. **Transcript Acquisition**: Using the YouTube API, the agent retrieves video information and autogenerated transcripts.
3. **Time-Based Segmentation**: Transcripts are segmented into meaningful parts without disrupting the narrative flow.

## Checkpoints

### Checkpoint 1: Transcript Processing
- The agent runs in the background and is triggered by a video ID.
- Logs the following:
  - Video metadata (ID, title, description).
  - Processed transcript (time-segmented).
  - Highlight snippets (empty at this stage).
- Uses Google Sheets for structured logging of requests and responses.

### Checkpoint 2: Highlight Analysis
- Scores highlight-worthiness using GPT-4.
- Produces a ranked list of top segments with engagement scores and timestamps.

## Development Tools
### Setting Up the Development Environment

**Create a Virtual Environment**:
Create a virtual environment to isolate dependencies.
```bash
python -m venv venv
```

**Activate the Virtual Environment**:
- On **Windows**:
  ```bash
  venv\Scripts\activate
  ```
- On **macOS/Linux**:
  ```bash
  source venv/bin/activate
  ```

**Install Dependencies**:
Install the required Python packages using the `requirements.txt` file.
```bash
pip install -r requirements.txt
```

### Running the API

To run the FastAPI application locally, use the following command:
```bash
uvicorn main:app --reload
```
This will start the server at `http://127.0.0.1:8000`, and the `--reload` option will auto-reload the server when code changes are detected.


### 7. Testing the API Endpoint

You can test the API endpoint by visiting the URL in your browser or using `curl` or any HTTP client:

```bash
curl http://localhost:8000/process_video/YOUR_VIDEO_ID_HERE
```
